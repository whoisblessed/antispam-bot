# Telegram Antispam Bot

Бот для защиты Telegram-чатов от спама. Верифицирует новых участников и фильтрует сообщения по ключевым словам с проверкой через LLM.

## Возможности

- Верификация новых участников через inline-кнопку или команду `/verify` с таймером 3 минуты
- Двухуровневая фильтрация сообщений: стоп-слова + проверка через LLM
- Два режима при обнаружении спама: удалить сообщение и заблокировать пользователя, или уведомить администратора
- Поддержка топиков (тредов) в супергруппах
- Логирование спам-событий в файл `spam.log`

## Установка

```bash
pip install -r requirements.txt
```

## Настройка

Скопируй `.env.example` в `.env` и заполни переменные:

```bash
cp .env.example .env
```

| Переменная | Описание |
|---|---|
| `BOT_TOKEN` | Токен бота от @BotFather |
| `ACTION_MODE` | `delete` — удалить спам и заблокировать, `notify_admin` — уведомить админа |
| `ADMIN_USERNAME` | Имя администратора без @ (для режима `notify_admin`) |
| `VERIFY_TIMEOUT` | Время на верификацию в секундах (по умолчанию 180) |
| `LLM_API_KEY` | API-ключ для LLM |
| `LLM_BASE_URL` | Базовый URL API (OpenAI-совместимый) |
| `LLM_MODEL` | Название модели |

## Настройка бота в Telegram

1. Создать бота через @BotFather, получить токен
2. Отключить privacy mode: @BotFather → `/setprivacy` → выбрать бота → Disable
3. Добавить бота в группу как администратора с правами: удалять сообщения, банить участников

## Как это работает

**Верификация новых участников**

При входе нового пользователя бот отправляет сообщение с inline-кнопкой и запускает таймер на 3 минуты. Если пользователь нажимает кнопку или отправляет `/verify` — верификация пройдена, сообщение бота удаляется. Если таймер истекает — бот банит пользователя и сразу разбанивает (удаление без постоянной блокировки).

**Фильтрация сообщений**

Каждое текстовое сообщение проверяется по списку стоп-слов. Если совпадение найдено — текст отправляется в LLM с вопросом "является ли это спамом?". Решение принимается на основе ответа модели. Если LLM недоступна или не отвечает за 8 секунд — сообщение считается спамом по факту совпадения стоп-слова. Администраторы чата проверке не подвергаются.

**Топики**

При фильтрации сообщений бот читает `message_thread_id` и отправляет ответ в тот же топик. При входе нового участника определить топик невозможно — Telegram не передаёт эту информацию в событии `chat_member`, поэтому верификационное сообщение отправляется в основной чат.
